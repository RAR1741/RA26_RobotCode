This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build & Deploy Commands

```bash
./gradlew build                # Build the project
./gradlew deploy               # Deploy to RoboRIO
./gradlew simulateJava         # Run robot simulation
./gradlew test                 # Run tests
```

All commands use the Gradle wrapper (`gradlew`). Team number is configured in `.wpilib/wpilib_preferences.json`.

## Architecture

This is an FRC 2026 competition robot (Team RAR1741) using WPILib's command-based framework with AdvantageKit logging (`LoggedRobot`).

### Key Layers

- **`Robot.java`** — Extends `LoggedRobot`, sets up AdvantageKit logging infrastructure
- **`RobotContainer.java`** — Wires subsystems together and binds controls
- **`controls/`** — `DriverControls` (port 0, drivetrain) and `OperatorControls` (port 1, mechanisms)
- **`subsystems/`** — Individual mechanisms, each a WPILib `SubsystemBase`
- **`generated/TunerConstants.java`** — Auto-generated by Phoenix Tuner X; do not hand-edit

### Subsystem Overview

- **`CommandSwerveDrivetrain`** — CTRE Phoenix 6 swerve drive (4 TalonFX modules + CANcoders), field-centric control, SysId routines, vision integration
- **`Superstructure`** — Orchestrator for all non-drivetrain subsystems (shooter, turret, intake, hopper, kicker). Coordinates aim sequences and stop-all commands.
- **`ShooterSubsystem`** — Dual SparkMax flywheel via YAMS `FlyWheel` abstraction, closed-loop RPM control
- **`IntakeSubsystem`** — Roller (ThriftyNova open-loop) + pivot arm (SparkMax with 125:1 gear reduction via YAMS `Arm`)
- **`TurretSubsystem`** — Ballistics calculator only (no motor control yet)
- **`KickerSubsystem` / `HopperSubsystem`** — Stubs, not yet implemented

### Vendor Libraries

| Library | Purpose |
|---------|---------|
| **CTRE Phoenix 6** | Swerve drivetrain (TalonFX, CANcoder) |
| **REVLib** | SparkMax motor controllers |
| **YAMS** | Mechanism abstractions (FlyWheel, Arm) |
| **AdvantageKit** | Structured logging and replay |
| **Maple Sim** | Physics simulation |

### Logging

Dual logging: NetworkTables (live) + WPILOGWriter (on-robot at `/home/lvuser/logs`). `Telemetry.java` publishes pose, speeds, module states, and Mechanism2d visualizations. AdvantageKit metadata tags the project as "RA26_RobotCode".

### Constants

All robot constants live in `Constants.java` with nested classes per subsystem. Motor CAN IDs for some subsystems are still placeholder (`99999`).
