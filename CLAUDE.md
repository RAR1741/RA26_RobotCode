# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build & Deploy Commands

```bash
./gradlew build                # Build the project
./gradlew deploy               # Deploy to RoboRIO
./gradlew simulateJava         # Run robot simulation
./gradlew test                 # Run tests
```

All commands use the Gradle wrapper (`gradlew`). Team number is configured in `.wpilib/wpilib_preferences.json`.

## Architecture

This is an FRC 2026 competition robot (Team RAR1741) using WPILib's command-based framework with AdvantageKit logging (`LoggedRobot`).

### Key Layers

- **`Robot.java`** — Extends `LoggedRobot`, sets up AdvantageKit logging infrastructure
- **`RobotContainer.java`** — Wires subsystems, binds controls, and sets up Choreo autonomous via `AutoChooser`
- **`controls/`** — `DriverControls` (port 0, drivetrain) and `OperatorControls` (port 1, mechanisms)
- **`subsystems/`** — Individual mechanisms, each a WPILib `SubsystemBase`
- **`subsystems/StateManager/`** — `StateManager` + `State` enum: zone-based robot state machine
- **`auto/`** — Choreo autonomous infrastructure (`AutoChooser`, `AutoMaker`, `ChoreoTraj`, `ChoreoVars`)
- **`commands/`** — `ShootOnTheMoveCommand`
- **`util/`** — `Zone` (polygon containment helper)
- **`wrappers/`** — `REVThroughBoreEncoder` (DIO-based absolute encoder wrapper)
- **`generated/TunerConstants.java`** — Auto-generated by Phoenix Tuner X; do not hand-edit

### Subsystem Overview

- **`CommandSwerveDrivetrain`** — CTRE Phoenix 6 swerve drive (4 TalonFX modules + CANcoders) on a `"Drivetrain"` CAN bus, field-centric control, SysId routines, vision integration
- **`Superstructure`** — Orchestrator for all non-drivetrain subsystems. Instantiates and coordinates shooter, turret, hood, intake, hopper, kicker, both Limelights, and `StateManager`. Exposes `isReadyToShoot` trigger (shooter + turret + hood all at target and valid state).
- **`StateManager`** — Singleton zone-based state machine. Tracks `inDecapitationZone` (dynamic box around nearest trench, scaled by robot velocity) and five pass zones. States: `SHOOTING`, `PASS_RIGHT_SIDE`, `PASS_LEFT_SIDE`, `PASS_DEAD_ZONE`.
- **`ShooterSubsystem`** — Dual TalonFX (KrakenX60, leader + follower) flywheel via YAMS `FlyWheel`, closed-loop RPM control
- **`HoodSubsystem`** — TalonFX (KrakenX44) pivot via YAMS `Pivot`, 40–80° range, homing sequence, decapitation-zone soft angle clamp. Depends on `StateManager`.
- **`TurretSubsystem`** — SparkMax (NEO, 25:1 reduction) via YAMS `Pivot`, ±FOV soft limits. Two REV Through Bore encoders on DIO (m12T / m13T) with Chinese Remainder Theorem rezero on connect. Auto-rezeroes when absolute encoders become available.
- **`IntakeSubsystem`** — Dual SparkFlex rollers (primary + secondary, NeoVortex) + SparkMax pivot arm (~19.6:1 gear reduction) via YAMS `Arm`. REV Through Bore encoder on DIO for absolute pivot position seeding. Stall detection triggers retract/redeploy ("hammer time") sequence.
- **`KickerSubsystem`** — SparkFlex (NeoVortex) flywheel via YAMS `FlyWheel`, 1000 RPM feed/eject
- **`HopperSubsystem`** — SparkMax (NEO) flywheel via YAMS `FlyWheel`, 4:1 reduction, 4000 RPM feed/eject
- **`LimeLightSubsystem`** — Two instances (`limelight-up`, `limelight-down`). MegaTag2 pose estimation with retry logic (up to 5 retries). Injects robot heading/angular-velocity for ExternalImu mode when enabled; uses InternalImuMT1Assist when disabled.

### Vendor Libraries

| Library | Purpose |
|---------|---------|
| **CTRE Phoenix 6** | Swerve drivetrain (TalonFX, CANcoder) |

| **YAMS** | Mechanism abstractions (FlyWheel, Arm, Pivot) |
| **YALL** | Yet Another Logging Library (YAMS telemetry support) |
| **AdvantageKit** | Structured logging and replay |
| **Maple Sim** | Physics simulation |
| **ChoreoLib** | Autonomous path following with Choreo trajectories |

### Logging

Dual logging: NetworkTables (live) + WPILOGWriter (on-robot at `/home/lvuser/logs`). `Telemetry.java` publishes pose, speeds, module states, and Mechanism2d visualizations. AdvantageKit metadata tags the project as "RA26_RobotCode".

### Autonomous

Choreo-based: `ChoreoTraj` enumerates all trajectories from the deploy directory. `RobotContainer` populates an `AutoChooser` (lazy-loading `SendableChooser`) published to SmartDashboard. Named events are wired in `buildTrajectoryRoutine()`.

### Constants

All robot constants live in `Constants.java` with nested classes per subsystem (e.g., `ShooterConstants`, `TurretConstants`, `HoodConstants`, `IntakeConstants`, `KickerConstants`, `HopperConstants`, `LimelightConstants`, `VisionConstants`, `StateConstants`). Field layout constants are in `FieldConstants.java`.
